# Правила разработки кода

## Общие принципы

- **KISS**: Выбирай самое простое решение. Избегай абстракций и паттернов, если они не критичны.
- **YAGNI**: Реализуй только то, что нужно сейчас. Не добавляй функционал "на будущее".
- **MVP**: Минимальный рабочий код важнее идеальной архитектуры.

## Стиль кода

- Используй стандартные возможности Python без излишних зависимостей.
- Синхронный код (без async/await).
- Простые структуры данных: словари, списки. Без сложных классов, если можно обойтись функциями.
- Имена переменных и функций на русском языке, если это улучшает читаемость бизнес-логики.

## Обработка ошибок

- Обрабатывай ошибки API (Telegram, GigaChat) с понятными сообщениями в логах.
- Не падай молча — логируй все ошибки в человекочитаемом формате.

## Логирование

- Все сообщения логов должны быть понятны обычному человеку.
- Используй стандартный модуль `logging`, вывод в консоль.
- Логируй ключевые события: распознавание места, нажатие кнопки, выбор победителя.

## Управление состоянием

- Храни активные розыгрыши в простом словаре в памяти.
- Обязательно очищай словарь после завершения розыгрыша или истечения таймера.
- Проверяй лимит в 5 активных розыгрышей перед созданием нового.

## Конфигурация

- Все настройки читай из `.env` через `python-dotenv`.
- Не хардкодь значения — используй конфигурацию с разумными дефолтами.

## Тестирование

- Пиши простые тесты для критичной логики (выбор победителя, распознавание места).
- Не тестируй тривиальный код.
